# 05 — UI, HUD, and Menus

## Mission

Bring the game presentation up to the premium arcade standard.

## HUD requirements

Display a clean readable HUD with some or all of:

- health bar or hearts
- XP / level bar
- score
- room number / dungeon depth
- current weapon or ability label
- pause button on mobile

## UI style rules

Preserve the template feel:

- large readable text
- rounded corners
- gradients and depth
- subtle glow accents
- dark UI panels with colourful highlights
- big touch-friendly buttons

## Menu screens

Polish these screens:

### Main menu
- title
- flavour subtitle
- play button
- control help
- optional mute toggle

### Pause panel
- resume
- restart
- back to menu

### Game over panel
- clear result summary
- retry button
- back to menu
- playful defeat copy

### Victory panel
- celebratory message
- result summary
- replay button

## Readability priorities

Make sure:

- HUD never blends into the playfield
- health changes are obvious
- upgrade choices are clear at a glance
- mobile buttons are not cramped
- text remains readable on phone screens

## Upgrade modal polish

If upgrade choices are present, make them feel exciting:

- animate the appearance
- give each option an icon or symbol
- show concise benefit text
- use celebratory particles or shimmer

## Success check

The game should now look intentional and polished even when sitting idle on the menu or game over screen.
