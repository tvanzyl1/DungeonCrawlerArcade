# 07 — Juice, Audio, and Final Polish

## Mission

Push the build from functional to delightful.

## Required polish pass

Add high-value feedback to important actions.

### On attack
- slash trail or projectile flare
- light flash or glow
- satisfying cadence

### On hit
- sparks
- impact pulse
- floating damage text
- small hit stop if appropriate

### On enemy death
- burst / poof / bone scatter / goo splat depending on enemy
- coin or gem pop-out
- satisfying disappearance animation

### On pickup
- bounce or magnet movement
- sparkle
- value pop text

### On player hurt
- strong but readable feedback
- brief flash
- subtle shake
- health UI reaction

### On room clear / victory
- celebratory pulse
- confetti-like magic burst or treasure sparkle
- triumphant overlay treatment

## Audio direction

If adding audio, keep it lightweight and arcade-friendly.

Suggested sound types:
- sword swish
- magic pop
- coin pickup
- hurt ping
- enemy defeat burst
- menu click
- victory sting

Keep sounds short, clean, and satisfying.

## Final consistency pass

Check that:

- all screens match the same visual language
- fantasy theme is consistent
- no leftover zombie references remain
- desktop and mobile controls both feel good
- first minute of play feels exciting
- failure is still entertaining

## Stretch goals

If time permits:

- add a boss room
- add a second weapon archetype
- add a treasure chest intermission
- add daily-run style seed display
- add small lore flavour lines between floors

## Definition of done

The build is done when it feels like a small premium browser arcade game someone would happily share or host online.
